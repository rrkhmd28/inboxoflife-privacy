# InboxOfLife プライバシーポリシー

**最終更新日**: 2026-05-28
**対象アプリ**: InboxOfLife（iOS、App Store 配信）
**Languages**: 日本語（このページ） / [English](./en.html)

---

## 1. はじめに

InboxOfLife（以下「本アプリ」）は、Ruriko（以下「開発者」）が個人で開発・運営するライフログアプリです。本ポリシーは、本アプリが取り扱うユーザー情報について説明します。

本アプリは**ローカルファースト**設計です。ユーザーが入力した情報（タスク、習慣、予定、買い物、Wishlist、メモ、体調メモ、活動ログ、チャット履歴）は、原則として端末内のローカルデータベースにのみ保存されます。

## 3. 収集する情報

### 3.1 端末内に保存される情報

以下の情報は、ユーザーの端末内（SQLite データベース、WatermelonDB）にローカル保存されます。**開発者および第三者は、これらの情報に直接アクセスできません**（端末からクラウドへのバックアップを除く）。

- ユーザーが入力したタスク・予定（日時）・習慣・買い物リスト・Wishlist・メモ・体調メモ・活動ログ
- 入力時刻・更新時刻
- アプリ内のチャット履歴（過去の入力と分類結果）
- アプリ設定（通知設定、ホーム画面の表示優先度など）

これらのデータは、ユーザーが本アプリを端末から削除すると同時に削除されます。

### 3.2 サーバーに送信される情報（AI 分類のため）

ユーザーがチャット入力欄に投稿した自然文を AI が分類するため、以下の情報が開発者の運用するサーバー（Railway 上）に送信されます。

- ユーザーが入力した自然文テキスト
- AI 分類の精度向上のための最小限の文脈情報:
  - 直近のチャット会話の数件
  - 直近に保存したエントリの参照情報（タイトル・カテゴリ・ID）
  - 開いている習慣・買い物・タスク・Wishlist のタイトル一覧（既存項目の完了報告を認識するため）

このサーバーはリクエストを受け取ったあと、後述の第三者 AI サービス（Google Gemini / OpenAI）に中継し、結果をユーザーの端末に返します。**サーバー側ではリクエスト本文を永続化しません**。

### 3.3 サーバー側ログ

ホスティング事業者（Railway）の標準仕様により、以下のメタデータがアクセスログとして短期間（おおむね 30 日）保持されます。

- IP アドレス
- リクエスト日時
- HTTP ステータスコード
- エンドポイントパス

これらは障害調査・不正アクセス検知のためにのみ参照され、外部に提供することはありません。

### 3.4 収集しない情報

本アプリは以下の情報を**収集しません**。

- アカウント情報（本アプリはログイン機能を持ちません）
- 氏名・電話番号・住所
- 連絡先（アドレス帳）
- 位置情報（GPS）
- 写真・カメラ
- HealthKit / Apple Health データ
- 広告識別子（IDFA）
- アプリ内行動の解析・トラッキング
- クラッシュレポート（本バージョンでは未導入）

## 4. 第三者サービス

本アプリは以下の第三者サービスを利用しています。

| サービス | 用途 | 送信されるデータ |
|---|---|---|
| Google Gemini API | 入力テキストのカテゴリ分類（主） | 入力文・文脈窓（§3.2） |
| OpenAI API | 入力テキストのカテゴリ分類（Gemini 失敗時のフォールバック） | 入力文・文脈窓（§3.2） |
| Railway | AI 分類中継サーバーのホスティング | リクエスト本文（通過）、アクセスログ（§3.3） |
| Apple App Store | アプリ配信 | Apple 標準の配信メタデータ |

各社のプライバシーポリシー:

- Google: <https://policies.google.com/privacy>
- OpenAI: <https://openai.com/policies/privacy-policy>
- Railway: <https://railway.com/legal/privacy>
- Apple: <https://www.apple.com/legal/privacy/>

## 5. 情報の利用目的

収集した情報は以下の目的でのみ利用します。

1. ユーザーが入力した内容を AI で分類し、適切なリストやカレンダーに振り分けるため
2. 端末内データベースに記録し、ユーザーが後から閲覧・編集できるようにするため
3. アクセスログによる障害調査および不正アクセスの検知

開発者が広告配信・マーケティング・第三者への販売の目的で利用することは**ありません**。

---

## 6. データの保持と削除

- **端末内データ**: ユーザーが本アプリを端末から削除すると、同時にすべて削除されます。アプリ内の「設定」画面からも個別データの削除が可能です。
- **サーバー側**: AI 分類のリクエスト本文は永続化しません。Railway のアクセスログは事業者の標準保持期間に従います。
- **第三者 AI**: 送信先である Google および OpenAI のデータ保持ポリシーに従います（上記リンク参照）。

---

## 7. データのエクスポート

ユーザーは、アプリ内の「設定」画面から端末内のすべてのデータを JSON 形式でエクスポートできます。これにより、いつでも自分のデータを取り出すことが可能です。

## 9. セキュリティ

- AI 分類サーバーへの通信はすべて HTTPS で暗号化されています。
- サーバーは共有トークンによる認証を要求し、想定外のクライアントからのアクセスを防いでいます。
- サーバー側でユーザー入力を永続化しないことで、データ漏洩のリスクを最小化しています。

ただし、インターネットを介する通信に絶対安全はありません。重要な機密情報は本アプリに入力しないことをお勧めします。
